# Case Studies — Wins and Failures

Use these to anchor recommendations in real outcomes. The throughline: **winners have an unreplicable per-page data moat; losers swap variables in a template.**

## Wins

### Zapier — the canonical win
- **~50,000+ integration pages** across three levels: app profile pages (`zapier.com/apps/[app]`), two-app integration pages, multi-app workflow pages.
- Pulls from proprietary data on **6,000–7,000+ apps**; partners submit content under editorial guidelines.
- App/integration pages drive **~15% of organic traffic (~235K monthly visits)**; the company quadrupled organic to **~5–6M+ monthly visits.**
- Success factors: real structured data, transactional design with CTAs, strong **hub-and-spoke** internal linking (app directory → integration pages), quality backlinks.

### Zillow
- Each property page tied to **unique MLS / tax / school / walkability data** + interactive tools (mortgage calc, maps).
- Unreplicable per-page data = **textbook moat.**

### NerdWallet — win *and* cautionary tale
- Proprietary calculators + real-time rate data + human-authored guidance; **$150M+ Q2 2024 revenue.**
- But Ahrefs documented **diminishing returns** and dependence on syndication/PR for backlinks — cautionary on **scale limits**: page count can rise while traffic stays flat.

### Tripadvisor
- "Things to do in [location]" + restaurant/hotel pages; value from **millions of UGC reviews** (226M+ monthly visits).

### G2
- Programmatic software **comparison / "vs" pages.**

### Canva
- Proprietary **template library** + usage analytics + human categorization.

## Failures (the patterns to avoid)

- **ZoomInfo** — database SEO boom-bust; scaled, then collapsed after spam updates.
- **Generic "best [category] for [use case]" pages** with no unique research.
- **"[service] in [city]" with only city-name swaps** — deindexed in March 2024 / 2026.
- One documented 2026 program had **8,000+ pages, of which only 312 ranked** by month three.

## What the contrast teaches

Every winner answers a *distinct query* with *data a competitor can't reproduce* and exposes it as clean, linked, server-rendered HTML. Every failure produces near-identical pages with no information gain. The method (templates, AI, automation) is never the problem; the **absence of unique value per page** always is.
